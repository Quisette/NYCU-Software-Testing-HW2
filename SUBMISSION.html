Part 2: Final Calc.js Implementation

File: Calc.js

class Calc {
    static add(a, b) {
        return a + b;
    }

    static subtract(a, b) {
        return a - b;
    }

    static multiply(a, b) {
        return a * b;
    }

    static divide(a, b) {
        if (b === 0) {
            throw new Error('Division by zero is not allowed');
        }
        return a / b;
    }
}

module.exports = Calc;

Phase 3: DIVIDE - Red → Green → Refactor

RED: Create Failing Tests

Created 9 test cases with critical design decisions:

Design Decision 1: Division Returns Floating-Point

test('should divide and return floating point result', () => {
  const result = Calc.divide(10, 3);
  assert.ok(Math.abs(result - 3.3333333) < 0.0001,
    `Expected ~3.3333333, got ${result}`);
});

This test REQUIRES division to return 3.3333..., not 3. This determines the behavior: floating-point division, not integer division.

Design Decision 2: Division by Zero Throws Error

test('should throw error when dividing by zero', () => {
  assert.throws(() => {
    Calc.divide(10, 0);
  }, Error);
});

This test REQUIRES an error to be thrown when dividing by zero.

Other test cases include:

  • Basic division (10 ÷ 2 = 5)
  • Negative numbers (-10 ÷ -2 = 5)
  • Mixed signs (10 ÷ -2 = -5)
  • Zero dividend (0 ÷ 5 = 0)
  • Decimal operands (10.0 ÷ 2.5 = 4.0)
  • Large numbers (1,000,000 ÷ 1,000 = 1,000)
  • Non-integer results (7 ÷ 2 = 3.5)

Initial Result: Tests FAIL with error: Calc.divide is not a function

GREEN: Implement Divide with Error Handling

static divide(a, b) {
    if (b === 0) {
        throw new Error('Division by zero is not allowed');
    }
    return a / b;
}

Implementation Details:

  1. Check if divisor is zero
  2. Throw error if zero (satisfies division by zero test)
  3. Return natural division result (JavaScript's / operator returns floats)

Result: All 9 divide tests PASS ✅

REFACTOR

No refactoring needed - error handling is explicit and minimal.
